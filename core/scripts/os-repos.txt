# scripts/os-repos.txt — the canonical OS-repo fleet that vendors Core.
# ──────────────────────────────────────────────────────────────────────────────
# ONE list of the repos Core fans out into, so adding a new OS target is a single
# edit here instead of a hardcoded array buried in sync-core.sh (B9). Read by
# scripts/sync-core.sh as the default TARGETS when no repos are named on the CLI.
#
# Format: one repo directory name per line. Blank lines and `#` comments ignored.
# Order is the fan-out order. Keep it case-insensitively alphabetical after
# dotfiles-MacBook (the reference implementation), which stays first so a smoke sync
# hits it soonest — i.e. Alpine sorts before Arch ('l' before 'r').
#
# DELIBERATELY ABSENT — not a Core-vendoring target (do not re-add):
#   dotfiles-Windows — the native Windows host layer is replicated from scratch
#     (PowerShell; not a one-to-one port of the Unix Core), so it carries no
#     vendored core/ subtree and sync-core.sh must never fan out into it.
#   dotfiles-Debian — was planned, never created, and is no longer being pursued.
#     The Debian family is covered by dotfiles-Kali (apt, Debian-family). Do not
#     re-add unless a standalone Debian OS repo is actually stood up.
# ──────────────────────────────────────────────────────────────────────────────

dotfiles-Alpine
